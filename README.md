# Embedded Systems Assignment

## Overview

This repository contains solutions for the Embedded Systems Assignment consisting of two tasks:

1. STM32G070RB Switch & LED Blinking
2. Producer-Consumer Sensor Buffer Simulation in C

Both implementations are written in C and demonstrate embedded programming concepts such as GPIO handling, interrupts, timers, circular buffers, synchronization, and power optimization.

---

# Project 1: STM32G070RB Switch & LED Blinking

## Objective

Develop firmware for the STM32 NUCLEO-G070RB board where a push button controls the LED blinking frequency.

### Functional Requirements

| Switch Press | LED State |
|--------------|-----------|
| 1st | Blink at 0.5 Hz |
| 2nd | Blink at 1 Hz |
| 3rd | Blink at 2 Hz |
| 4th | LED OFF |
| 5th | Cycle repeats from 0.5 Hz |

---

## Features

- GPIO configuration using STM32 HAL
- External Interrupt (EXTI) for push button
- Timer interrupt for LED blinking
- Debouncing using software
- Low-power optimization using Sleep Mode
- Modular and well-commented code

---

## Hardware Used

- STM32 NUCLEO-G070RB
- On-board User LED
- On-board Push Button
- STM32CubeIDE
- STM32 HAL Drivers

---

## Power Optimization

To reduce power consumption:

- CPU enters Sleep Mode while waiting for interrupts.
- LED toggling handled by timer interrupt.
- Button press handled using EXTI interrupt.
- CPU wakes only when required.

# Project 2: Producer-Consumer Sensor Buffer Simulation

## Objective

Simulate a sensor generating random data every second while another task periodically consumes data from a shared buffer.

---

## Functional Requirements

Producer:

- Executes every simulated second.
- Generates 0–5 random bytes.
- Stores generated bytes into a shared circular buffer.

Consumer:

- Executes every simulated 10 seconds.
- Checks whether at least 50 bytes are available.
- Prints only the latest 50 bytes in hexadecimal.
- Removes printed bytes from the buffer.

---

## Features

- Circular buffer implementation
- Producer-Consumer architecture
- Random sensor data generation
- Timer simulation
- Hexadecimal output
- Memory-safe implementation

---

## Assumptions

- 1 simulated second = 200 ms (to reduce execution time).
- Buffer is large enough to store generated data.
- Single producer and single consumer.
- No operating system or multithreading is used.
- Standard C library functions are available.
- Random numbers generated using `rand()`.

---

## Example Output

```
Time 1s:
Producer generated 4 bytes

Time 10s:
Buffer contains 39 bytes
Not enough data

Time 20s:
Latest 50 Bytes:
A2 17 3C FF 90 ...

Printed bytes removed.
```

# How to Build

## STM32 Project

1. Open the project in STM32CubeIDE.
2. Build the project.
3. Flash to the STM32 NUCLEO-G070RB board.
4. Press the user button to observe LED frequency changes.

---

## Producer-Consumer Simulation

Compile using GCC:

```bash
gcc producer_consumer.c -o producer_consumer
```

Run:

```bash
./producer_consumer
```

or execute directly on online C compilers like Programiz.
